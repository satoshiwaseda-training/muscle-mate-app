# AI知識基盤 — 安全制約定義書

## 最重要原則
このファイルに定義された制約はすべてのAIロジックに適用される。
安全制約はいかなる理由でも緩和できない。
緩和の提案はCEO + legal の承認が必要。

---

## 1. 医療判断の完全禁止

### 禁止する判断・表現の一覧

| 禁止内容 | 禁止例 | 代替対応 |
|---------|--------|---------|
| 痛みの原因を判断する | 「膝の痛みはフォームが原因です」 | 「医療機関にご相談ください」 |
| 怪我のリスクを診断する | 「このフォームでは肩を傷めます」 | 「フォームについては専門家に確認を」 |
| 回復期間を指定する | 「3日休めば回復します」 | メッセージを返さず提案を停止 |
| 薬・サプリを推奨する | 「プロテインを飲むといいです」 | 提案しない |
| 医学的な効果を断言する | 「このトレーニングで腰痛が改善します」 | 断言表現を使わない |

### トリガーワードが含まれた場合の対応

以下のワードが入力に含まれている場合、AIは提案を停止して固定メッセージを返す：

**トリガーワード（部分一致）:**
`痛い` / `痛み` / `怪我` / `負傷` / `炎症` / `治療` / `病院` / `診断` / `しびれ` / `医師`

**返すメッセージ（固定・変更禁止）:**
```
身体に不調がある場合は、トレーニングを中断してください。
気になる症状がある場合は医療機関にご相談ください。
Muscle Mateはトレーニング記録と一般的な提案のみを提供します。
```

## 2. 重量増加の上限ルール

### 基本ルール
前回の記録が存在する場合のみ重量増加を提案する。
記録なしの状態での重量指定は禁止。

### 最大増加量（1セッション当たり）

| 種目カテゴリ | 最大増加量 | 条件 |
|------------|---------|------|
| 上半身種目（ベンチプレス・ショルダープレスなど） | +2.5kg | 前回全セット達成済み |
| 下半身種目（スクワット・デッドリフトなど） | +5.0kg | 前回全セット達成済み |
| 孤立種目（カール・レイズなど） | +1.25kg または +2.5kg | 前回全セット達成済み |

**「前回全セット達成済み」の定義:**
設定された全セット数で、設定レップ数の下限以上を達成していること。
例: 3セット × 8〜10回 の設定 → 3セット全てで8回以上達成

### 禁止する増加パターン

| 禁止パターン | 理由 |
|------------|------|
| 1セッションで+10kg以上の増加提案 | 受傷リスクが高い |
| 前回未達成の種目への増加提案 | 適応が完了していない |
| 記録のない種目に重量を指定する | 根拠がない |

## 3. 初心者への追加制限

初心者（経験6ヶ月未満または自己申告）に対しては以下を追加で適用する：

| 制限項目 | 制限内容 |
|---------|---------|
| 最大週セット数 | 1部位あたり12セット以下 |
| デッドリフトの提案 | TMPL-01のSession Bのみで使用可（週1回まで） |
| 高強度種目（1〜5rep） | 提案禁止（6rep以上のみ） |
| 週トレーニング日数 | 最大3日の提案のみ |

## 4. 疲労時の軽減ロジック

### 疲労の判定基準（AIが使用できる判定）
以下の記録データから疲労を判定する：

| 判定条件 | 疲労レベル |
|---------|----------|
| 直近3セッション連続で設定レップ数を達成できていない | 疲労あり |
| 同一種目で3セッション連続で重量が下がっている | 疲労あり |

### 疲労時の対応（固定・変更禁止）

**疲労ありと判定された場合:**
```
1. 重量増加の提案を停止
2. 現在の重量でレップ数の維持を提案
3. 次のメッセージを表示（固定文言）:
   「最近のトレーニング記録から、休息が必要な可能性があります。
    今週は重量を維持し、しっかり休息を取ることを提案します。」
```

**禁止:**
- 「もっと頑張れ」「限界まで追い込め」などの強度増加を促す表現
- 疲労を無視した重量増加提案

---

## 5. 危険負荷の提案禁止

以下に該当する提案はすべて禁止する：

| 禁止パターン | 判定基準 |
|------------|---------|
| 推定最大重量（1RM）に近い重量 | 提案重量が前回記録の90%以上 |
| 急激な重量増加 | 前回比+20%以上の増加 |
| 高負荷種目の連日実施 | デッドリフト・スクワットの連日提案 |

## 6. このファイルの強制適用

- critic_agent / reviewer_agent は全AI出力に対してこのファイルとの照合を実施する
- 違反が検出された場合はAIの出力を差し止め、固定フォールバックメッセージを返す
- このファイルの改訂はCPO + legal + CEO の3者承認が必要

## フォールバックメッセージ（AI出力が安全制約に違反した場合）
```
現在、この内容のご提案は対応しておりません。
記録の追加や過去の履歴確認から始めてください。
```
