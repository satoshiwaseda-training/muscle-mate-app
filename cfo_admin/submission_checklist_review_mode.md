# CFO / legal 成果物 — App Store提出チェックリスト

## 目的
審査提出前に全条件が満たされていることを1ファイルで確認する。
全項目に「✅」が揃うまで提出しない。

---

## STEP 1: 法務・規約（legal確認）

### 利用規約
- [ ] 利用規約が設定画面に表示される（静的テキスト・WebView不使用）
- [ ] 利用規約に以下の記述が含まれている:
  - サービスの概要（筋トレ記録アプリである旨）
  - データの取り扱い（ローカル保存のみ・外部送信なし）
  - 免責事項（トレーニング内容に関する医療的アドバイスを行わない旨）
  - 問い合わせ先（メールアドレス）
  - 改訂方針

### プライバシーポリシー
- [ ] プライバシーポリシーが設定画面に表示される（静的テキスト）
- [ ] App Store Connectの「プライバシーポリシーURL」欄に有効なURLが設定されている
  - ※ URL必須（App Storeの仕様）。GitHubの静的ページ等でも可
- [ ] プライバシーポリシーに以下の記述が含まれている:
  - 収集するデータ: なし（ローカルに保存するトレーニング記録のみ）
  - 外部送信: 行わない
  - 第三者提供: 行わない
  - データ削除方法: アプリ内「設定 → 全データを削除」から実行可能
  - 問い合わせ先

### 問い合わせメール
- [ ] 設定画面の「お問い合わせ」タップでメールアプリが開く
- [ ] 宛先メールアドレスが正しく設定されている（実在するアドレス）

## STEP 3: 技術確認（CTO → CFO報告）

### ネットワーク
- [ ] アプリ実行中に外部へのHTTPリクエストが一切発生しない（Charlesで確認）
- [ ] Info.plist に `NSAllowsArbitraryLoads = false` が設定されている
- [ ] AndroidManifest.xml に `INTERNET` パーミッションが含まれていない

### 権限
- [ ] Info.plist に以下の権限記述が含まれていない:
  - NSCameraUsageDescription
  - NSMicrophoneUsageDescription
  - NSLocationWhenInUseUsageDescription / NSLocationAlwaysUsageDescription
  - NSPhotoLibraryUsageDescription
  - NSHealthShareUsageDescription / NSHealthUpdateUsageDescription
  - NSContactsUsageDescription
- [ ] アプリ実行中に権限リクエストダイアログが表示されない

### クラッシュ
- [ ] TestFlightでのクラッシュ件数: 0件
- [ ] 以下のシナリオで全てクラッシュしない:
  - [ ] 初回起動（DBが存在しない状態）
  - [ ] 記録追加（正常入力）
  - [ ] 記録追加（空入力でのバリデーションエラー）
  - [ ] 記録削除
  - [ ] 全データ削除（0件の状態での実行を含む）
  - [ ] アプリをバックグラウンドに移動 → 復帰
  - [ ] アプリを強制終了 → 再起動（データが保持されていること）

### SDK
- [ ] Firebase が含まれていない
- [ ] Analytics SDK が含まれていない
- [ ] Crashlytics が含まれていない
- [ ] 広告SDK（AdMob等）が含まれていない
- [ ] 決済SDK（RevenueCat等）が含まれていない

## 提出後の確認事項

提出後に審査チームから連絡が来た場合の対応方針:
- 返答期限: 連絡受信から48時間以内
- 返答担当: CEO（legal確認後）
- 返答言語: 日本語 + 英語（両方用意）
- 対応不可な要求（機能追加等）は「現バージョンでは対応しておらず、将来のアップデートで検討します」と回答
