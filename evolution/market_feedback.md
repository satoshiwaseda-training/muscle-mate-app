# evolution — 市場フィードバック定義書

## 目的
アプリ外の市場シグナル（競合・トレンド・ユーザー声・ストアレビュー）を
構造化して取り込み、内部KPIデータと組み合わせて仮説の精度を上げる。
外部シグナルなしの仮説は「自社の思い込み」になる。

---

## 取り込むシグナルの種類と取得頻度

| シグナル種別 | 取得方法 | 取得頻度 | 担当部門 |
|------------|---------|---------|---------|
| App Store / Google Playレビュー | ストア管理画面 / スクレイピング | 週次 | analytics |
| 競合アプリのアップデート情報 | 各ストアのバージョンノート | 週次 | CMO |
| SNSでの言及（X, Instagram） | ハッシュタグ検索 | 週次 | CMO |
| ユーザーインタビュー | CPOが設計・growthが実施 | 月次（最低5人） |  growth + CPO |
| 解約アンケート結果 | monetization管理 | 都度（解約時） | monetization |
| フィットネス業界レポート | 公開統計・業界誌 | 四半期 | CEO |
| Google Trendsデータ | 「筋トレ」「ダイエット」など | 月次 | analytics |

```

【シグナル内容（事実のみ）】
（解釈を含めずに、見聞きした内容をそのまま記述する）

【自社KPIへの影響予測】
（このシグナルが示唆する自社KPIへの影響を1〜2文で記述）

【推奨アクション】
（このシグナルを受けて取るべきアクションを1件・具体的に記述）

【hypothesis_engineへの連携】
仮説化: 要 / 不要
仮説化する場合の対象KPI: [KPI名]
```

## ストアレビュー分析フレームワーク（週次）

analyticsが毎週月曜に以下を実施する：

### Step 1: レビュー収集
- App Store / Google Playの直近7日間の新着レビューを全件取得
- 評価別に分類（★1〜2 / ★3 / ★4〜5）

### Step 2: テーマ分類
各レビューを以下のテーマに分類する（複数可）：

| テーマID | テーマ名 | behavior_modelのステップ |
|---------|---------|----------------------|
| THM-01 | UIが分かりにくい | 理解 |
| THM-02 | メニューに納得できない | 納得 |
| THM-03 | トレーニングが続けられない | 行動 |
| THM-04 | 達成感がない | 報酬 |
| THM-05 | リマインドが助かる / 邪魔 | 継続 |
| THM-06 | 料金に不満 | — |
| THM-07 | バグ・クラッシュ | — |
| THM-08 | 機能リクエスト | — |
| THM-09 | 競合比較（言及あり） | — |

### Step 3: ボリューム計測と変化検知
```
テーマ別レビュー数（直近7日） vs テーマ別レビュー数（前週）
変化率 ±20%以上のテーマをフラグ立て → evolutionへ即時通知
```

### Step 4: 週次サマリー出力
```
【ストアレビュー週次サマリー - YYYY/MM/DD】

総レビュー数: N件（★平均: X.X）
前週比: +N件 / 評価変化: +0.X

■ 急増テーマ（前週比+20%以上）
  - [テーマ名]: [N]件（前週[N]件）→ evolution通知済み

■ ネガティブレビュー上位テーマ（★1〜2）
  1. [テーマ名]: [N]件
  2. [テーマ名]: [N]件

■ ポジティブレビュー上位テーマ（★4〜5）
  1. [テーマ名]: [N]件

■ 競合言及
  - [競合アプリ名]: [N]件（内容: ○○が優れているとの声）
```

|---------|---------|---------|------|
| ※CMOが定義・四半期ごとに見直し | — | 週次 | CMO |

### 競合アップデートの評価基準
競合がアップデートした場合、以下の軸で評価する：

```
1. Muscle Mateにない機能か？ → ある場合: 仮説化を検討
2. behavior_modelのどのステップを強化している？
3. ユーザーレビューでの反応は？（取得から2週間後に評価）
4. 自社のD7/D30継続率への影響シグナルがあるか？
```

## ユーザーインタビュー設計（月次・最低5人）

### 対象ユーザーの選定基準
| インタビュー種別 | 対象 | 目的 |
|---------------|------|------|
| 継続ユーザーインタビュー | D30以上のアクティブユーザー | 継続理由・好きな機能の特定 |
| 離脱ユーザーインタビュー | 解約後30日以内のユーザー | 離脱理由・改善点の特定 |
| 新規ユーザーインタビュー | 登録後7日以内のユーザー | オンボーディングの課題特定 |

### 必須質問（変更禁止）
1. 「前回使ったのはいつですか？その時、何をしましたか？」（実際の行動）
2. 「アプリを使って一番良かった瞬間はどんな時ですか？」（報酬の特定）
3. 「アプリを使うのをやめようと思った瞬間はありましたか？」（離脱ポイント）
4. 「このアプリを友人に勧めるとしたら、何と言いますか？」（NPS・価値の言語化）
5. 「もし1つだけ変えられるとしたら何を変えますか？」（最重要改善点）

### インタビュー結果の記録
各インタビューはMKT-[ID]として記録し、learning_repositoryに連携する。
