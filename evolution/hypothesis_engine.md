# evolution — 仮説エンジン定義書

## 目的
最下位KPIを特定し、再現性のある仮説を自動生成する。
「感覚で試す」を組織から完全に排除する。

---

## 仮説の構造（必須フォーマット）

全仮説は以下の形式で記述する。フォーマットを満たさない仮説は実験対象にしない。

```
仮説ID: HYP-[YYYYMMDD]-[連番3桁]
対象KPI: [D1継続率 / D7継続率 / workout完了率 など]
現状値: [XX%]
目標値: [XX%（実験成功の定義）]

仮説文:
「[対象ユーザーセグメント] が [現状の課題行動] をしているため、
 [具体的な変更] を行うと、
 [対象KPI] が [X]% 改善すると予測する。」

根拠:
- [learning_repositoryの参照ID または analyticsのデータ]
- [market_feedback.mdの外部シグナル または ユーザーインタビュー]

実験設計（概要）:
- 変更内容: [具体的な変更箇所]
- 対象: [全ユーザー / 新規ユーザーのみ / Standardプランのみ など]
- 期間: [X日間]
- 判定指標: [KPI名と閾値]

コスト見積もり:
- 開発工数: [Xストーリーポイント]
- 実験コスト: [X円]
- CTO承認要否: [不要 / 要（理由）]
- CEO承認要否: [不要 / 要（理由）]
```

## 仮説生成アルゴリズム

### トリガー
- 週次レビューで最下位KPIが確定した直後に起動
- KPIがCriticalアラート（ops_realtime）を受けた直後に起動

### 生成ステップ

**Step 1: learning_repositoryを参照**
同一KPIに対して過去に実行した実験の結果を取得する。
- 成功実験: その施策を応用・拡張できないか確認
- 失敗実験: 同じ仮説の繰り返しを防ぐ（同一仮説の再提案は禁止）

**Step 2: behavior_model.mdと照合**
対象KPIが属するステップを特定する：

| KPI | behavior_modelのステップ |
|-----|----------------------|
| D1継続率 | 理解・行動・報酬 |
| D7継続率 | 報酬・継続 |
| D30継続率 | 継続 |
| workout完了率 | 行動 |
| 転換率 | 納得 |

そのステップで「ユーザーが止まっている理由」を3つ列挙する。

**Step 3: 仮説を3件生成**
以下の3つの切り口で必ず1件ずつ生成する：

| 切り口 | 説明 | 例 |
|--------|------|---|
| UI変更 | 表示・レイアウト・タイミングの変更 | 報酬画面の表示タイミングを0.5秒早くする |
| ロジック変更 | アルゴリズム・提案内容の変更 | メニュー提案をユーザー実績ベースにする |
| コミュニケーション変更 | 通知・コピー・メッセージの変更 | 完了翌日の通知文言を成果ベースにする |

**Step 4: 優先度スコアリング**
3件の仮説を以下の式でスコアリングし、順位付けする：

```
優先度スコア =
  (KPI影響予測 × 0.4)        // 高=10 / 中=5 / 低=2
+ (実装速度 × 0.3)           // 1週間以内=10 / 2週間=5 / 1ヶ月=2
+ (学習価値 × 0.2)           // 新しい知見=10 / 既存の応用=5 / 確認のみ=2
+ (リスクの低さ × 0.1)       // リスクなし=10 / 軽微=5 / 要注意=2
```

## 仮説バックログ管理

### ステータス定義
| ステータス | 定義 |
|-----------|------|
| PENDING | 生成済み・実験待ち |
| RUNNING | 実験中 |
| SUCCESS | 実験成功・本番適用検討中 |
| FAILED | 実験失敗・learning_repositoryに記録済み |
| ARCHIVED | 長期間未実行・現状と乖離のため廃棄 |

### バックログ更新ルール
- 毎週月曜のレビューで全PENDINGの優先度を再スコアリングする
- PENDING状態が8週間を超えた仮説はARCHIVEDに移動する
- 同一KPIへのPENDING仮説は最大5件まで。超過分はARCHIVED

## 禁止仮説パターン

以下の仮説は生成しない・実験しない：

| 禁止パターン | 理由 |
|------------|------|
| 「通知を増やす」（週3回超） | 継続率を下げるリスクが高い |
| 「課金を強制するペイウォール」 | ethics_policy.md違反 |
| 「個人情報をより多く収集する」 | legal部門の事前審査が必要 |
| 「AIが医療アドバイスをする」 | ethics_policy.md違反 |
| FAILED仮説と90%以上類似する仮説 | 同じ失敗の繰り返し |
