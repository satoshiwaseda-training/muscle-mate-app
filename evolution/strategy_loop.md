# evolution — 戦略ループ定義書

## 目的
Muscle Mateの継続率・LTVを最大化するための
意思決定サイクルを自動で回し続ける。
ループが止まることは組織の停止と同義。

---

## マスターループ（4週サイクル）

```
Week 1: 計測（Measure）
  analyticsの全KPIを収集・異常値を特定

Week 2: 仮説（Hypothesize）
  最下位KPIに対してhypothesis_engineが仮説を生成

Week 3: 実験（Experiment）
  優先度1位の仮説でauto_experimentを実行

Week 4: 学習（Learn）
  実験結果をlearning_repositoryに記録・次のサイクルに反映
```

**4週サイクルを常時1本以上並列で実行すること。**
サイクルのストップはCEO承認が必要。

---

## 週次レビュー（毎週月曜 10:00 実行）

### Step 1: KPIスコアカード更新
analyticsから以下を取得し、目標対比を更新する：

| KPI | 目標値 | 取得元 |
|-----|--------|--------|
| D1継続率 | 60% | analytics 週次レポート |
| D7継続率 | 40% | analytics 週次レポート |
| D30継続率 | 35% | analytics 月次レポート |
| workout完了率 | 80% | analytics 週次レポート |
| Free→Standard転換率 | 15% | monetization 月次レポート |
| AIコスト/MAU | 5円以下 | infra_cost 週次レポート |

### Step 2: 最下位KPI特定
目標対比で最も乖離が大きいKPIを1件選出する。

**乖離の計算式:**
```
乖離スコア = (目標値 - 実績値) ÷ 目標値 × 100
例: D7継続率 目標40% / 実績32% → 乖離スコア = 20%
```

### Step 3: 仮説の優先度更新
hypothesis_engineの仮説リストを最下位KPIベースで並び替える。

### Step 4: 実験ステータス確認
実行中の実験の中間数値を取得し、以下を判定する：
- 早期終了基準に達したか（auto_experiment.md 参照）
- 追加サンプルが必要か
- 問題なく継続か

### Step 5: 週次レポート出力
```
【evolution 週次レポート - YYYY/MM/DD】

最下位KPI: [KPI名] 実績[X]% / 目標[Y]% / 乖離[Z]%
実行中の実験: [実験ID] - [仮説名] (経過[N]日)
今週の判定: [完了した実験の結果サマリー] / なし
learning_repository 追加: [N]件
次週の優先仮説: [仮説ID] - [仮説名]
```

## 月次戦略レビュー（毎月1日 実行）

### 評価項目
1. **4週サイクルの完了率**: 計画した実験を完了できたか
2. **仮説の精度**: 成功率（成功実験数 ÷ 完了実験数）
3. **学習の質**: learning_repositoryに蓄積した知見が次の仮説に反映されているか
4. **市場シグナルの取り込み**: market_feedback.mdのデータが戦略に反映されているか

## ループの停止ルール

以下の状況のみ、CEOの承認を得てループを一時停止できる：
- 全KPIが目標値を上回っており、実験によるリスクが継続率を脅かす
- 重大なインシデント対応中でリソースが確保できない
- 法的問題により実験対象機能が停止中

停止期間は最大4週間。それ以上はCEOが再承認する必要がある。
