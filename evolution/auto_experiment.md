# evolution — 自動実験定義書

## 目的
hypothesis_engineが生成した仮説を、
バイアスなく・再現可能な方法で自動実験し、
数値で成否を判定する。
「なんとなく良さそう」を組織から排除する。

---

## 実験の種類と適用条件

| 実験種別 | 適用条件 | 承認要件 |
|---------|---------|---------|
| A/Bテスト（UI） | ユーザー分割可能・計測イベントあり | CPO承認 |
| A/Bテスト（通知） | プッシュ通知許可ユーザー対象 | CPO + CMO承認 |
| A/Bテスト（価格） | Standardプラン未加入ユーザー対象 | CPO + CFO + CEO承認 |
| シャドーテスト | 本番に影響させずAIロジックを並列実行 | CTO承認 |
| 段階ロールアウト | 新機能を5%→20%→50%→100%で展開 | CPO + CTO承認 |

---

## 実験設計テンプレート（必須）

```
実験ID: EXP-[YYYYMMDD]-[連番3桁]
仮説ID: HYP-[YYYYMMDD]-[連番3桁]

【実験設計】
対象ユーザー: [セグメント定義]
割り当て比率: Control [X]% / Treatment [Y]%
最小サンプル数: [N人]（統計的有意性 p<0.05・検出力 80%を保証する計算値）
実験期間: [開始日] 〜 [終了日]（最短7日・最長30日）

【計測指標】
プライマリKPI: [KPI名]（判定の主軸）
セカンダリKPI: [KPI名1]・[KPI名2]（副次的確認）
ガードレール指標: [KPI名]（この指標が悪化したら即時停止）

【成功基準】
プライマリKPI が Treatment > Control で [X]% 以上改善
かつ p値 < 0.05
かつ ガードレール指標の悪化が ±2% 以内

【早期停止基準】
以下のいずれかに該当した場合、実験を即時停止する：
- ガードレール指標が Control対比 -5% 以上悪化
- 実験起因のエラー率が 1% を超えた
- ops_realtimeがCriticalアラートを発報した

【実装内容】
Control: [現行の状態を具体的に記述]
Treatment: [変更後の状態を具体的に記述]
実装担当: CTO（executor_agent経由）
```

---

## 最小サンプル数の計算基準

実験ごとに以下の計算を必ず実施すること：

```
最小サンプル数（片側） =
  (Z_α + Z_β)² × p(1-p) / δ²

パラメータ定義:
  Z_α = 1.645（有意水準 α=0.05 片側検定）
  Z_β = 0.842（検出力 80%）
  p   = コントロール群のプライマリKPI現状値
  δ   = 検出したい最小改善幅（例: D7継続率で +3pt = 0.03）

例: D7継続率 現状36% / 最小改善幅+3pt
  = (1.645 + 0.842)² × 0.36 × 0.64 / 0.03²
  = 6.185 × 0.2304 / 0.0009
  ≈ 1,583人（片側）→ 両側で 3,166人
```

**サンプル数が7日間で達成できない場合は実験期間を延長する。最大30日。**

## 実験実行フロー

```
Step 1: 仮説ID受け取り（hypothesis_engineから）
  ↓
Step 2: 実験設計テンプレートを作成・承認申請
  （必要な承認者にドキュメントを送付）
  ↓
Step 3: CPO / CTO が実装を実行
  （planner_agent → executor_agentのフローで処理）
  ↓
Step 4: 実験開始（analyticsがイベント計測を開始）
  ↓
Step 5: 日次モニタリング（早期停止基準のチェック）
  ↓
Step 6: 判定実施（期間終了 または 最小サンプル達成後）
  ↓
Step 7: 結果をlearning_repositoryに記録
  ↓
Step 8: 成功の場合 → CPO + CMOに本番適用を提案
         失敗の場合 → 学習内容を次の仮説生成に反映
```

## 判定フォーマット（実験完了時に必ず記録）

```
【実験判定レポート】
実験ID: EXP-[YYYYMMDD]-[連番]
仮説ID: HYP-[YYYYMMDD]-[連番]
実験期間: [開始日] 〜 [終了日]（[N]日間）
有効サンプル数: Control [N]人 / Treatment [N]人

【数値結果】
プライマリKPI:
  Control: [X]% / Treatment: [Y]% / 差分: [+/-Z]pt / p値: [0.0XX]
セカンダリKPI:
  [KPI名]: Control [X] / Treatment [Y] / 差分: [+/-Z]
ガードレール指標:
  [KPI名]: Control [X] / Treatment [Y] → 問題なし / 要注意

【判定】
結果: 成功 / 失敗 / 判定不能（サンプル不足）
統計的有意性: あり（p=[0.0XX]） / なし（p=[0.XXX]）

【学習内容（learning_repositoryへの記録内容）】
- 学んだこと: [具体的に1〜3文]
- 次の仮説への示唆: [具体的に1文]

【次のアクション】
成功の場合: [本番適用の提案先と期限]
失敗の場合: [廃棄 or 変数を変えて再実験]
```
