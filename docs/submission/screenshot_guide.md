# スクリーンショット撮影手順書

App Store 提出に必要な **5 枚のスクリーンショット**を Mac 上のシミュレータで撮影する手順。提出計画書 v1.3 §5.3 準拠。

---

## 0. 必要なもの

- Mac（Apple Silicon または Intel）
- Xcode インストール済み（iOS Simulator 含む）
- 本リポジトリのローカルクローン
- バックエンドサーバー（ローカルまたは本番）

---

## 1. シミュレータの起動

### 1-1. ターミナルでアプリを起動

```bash
cd /Users/User/Desktop/Claude/muscle-mate-app/frontend
flutter run -d "iPhone 16 Pro Max" --release \
  --dart-define=API_BASE_URL=http://127.0.0.1:8000 \
  --dart-define=ENABLE_EXTERNAL_AI=false
```

iPhone 16 Pro Max シミュレータが起動し、アプリが開きます（初回起動なら自動でセットアップ）。

> 💡 デバイス名がわからないときは `flutter devices` で利用可能なシミュレータ一覧が表示されます。**6.9 inch ディスプレイ**（iPhone 16 Pro Max など）を選んでください。

### 1-2. シミュレータの解像度確認

App Store の許容サイズは：
- **1290 × 2796** （iPhone 16 Pro Max 標準）
- 1260 × 2736（iPhone 15/16 Pro Max）
- 1320 × 2868（iPhone 17 Pro Max・将来用）

iPhone 16 Pro Max シミュレータからの書き出しは自動的に 1290 × 2796 になります。**リサイズせずそのまま提出**してください。

## 2. ダミーデータの準備

撮影前に、リアルなダミーデータを入れて画面に説得力を持たせます。

### 2-1. 同意画面 → 設定

1. 起動 → 同意画面で「13 歳以上」にチェック → 「同意してはじめる」
2. オンボーディングは飛ばす（または基本入力）
3. ハンバーガーメニュー → 「設定」
4. 以下を入力：
   - レベル：**中級者**
   - 目的：**筋力アップ**
   - セッション時間：**45 分**
   - BIG3 MAX：ベンチ **80kg** / スクワット **100kg** / デッド **120kg**
   - 体重：**70kg**（任意）
   - 年齢：**32**（任意）
   - 使用器具：すべてチェック
5. 「設定を保存する」

### 2-2. ダミーのトレーニング記録 3 件作成

過去 3 日分の記録を入れて、履歴・ベスト・カレンダーを充実させます：

1. ホーム → 「+」→ 「メニューを提案してもらう」 → 生成 → セッション開始 → 1〜2 種目だけ重量・回数を入れて完了
2. 同じ流れを 2 回繰り返す（種目を変えて）

> 💡 シミュレータの日付を変えたい場合は、`xcrun simctl status_bar booted override --time "9:41"` で時刻だけ統一可能（バッテリー満タンも同コマンド）。

## 3. 5 枚のスクリーンショット撮影

各画面の説明と、撮影順番。

### SS-01：ホーム画面（カレンダー + 進捗）

1. ホームに戻る
2. カレンダーが表示される状態
3. 今日の日付に「炎マーク」が出ていれば理想（記録済み）
4. ハンバーガーメニューは閉じた状態

撮影：シミュレータがフォーカス → **Cmd + S**（書き出し）

### SS-02：メニュー提案結果

1. ホーム → 「+」→ 「メニューを提案してもらう」
2. 入力画面で「目標：筋力アップ」など適当に選んで「メニューを提案してもらう」
3. メニュー提案結果が表示された画面（「今日のメニュー」セクションが見える状態）
4. **論文ベースの提案根拠カード**が見えていれば最強

撮影：**Cmd + S**

### SS-03：ワークアウトセッション

1. メニュー提案画面で「このセッションを開始」
2. 種目入力画面で、最初の種目に重量・回数を入れて 1 セット完了
3. 進捗バーが少し進んだ状態
4. **熊のキャラクターが見える状態**で撮影

撮影：**Cmd + S**

### SS-04：履歴画面（部位ハイライト付き）

1. ハンバーガーメニュー → 「履歴」
2. 過去の記録カードのうち、**胸の日**などをタップ
3. 詳細モーダルが開いて、**熊が表示される**
4. 鍛えた部位がオレンジでハイライトされている

撮影：**Cmd + S**

### SS-05：実績シェア画面

1. ハンバーガーメニュー → 「実績をシェア」
2. 「1 週間」タブを選択（数字が入って見栄えが良い）
3. **「持ち上げた合計重量」+ 物理オブジェクト比較カード**が見える状態

撮影：**Cmd + S**

## 4. 書き出しと確認

### 4-1. 保存先

シミュレータで **Cmd + S** すると、デスクトップに自動保存されます。ファイル名は：
```
Simulator Screenshot - iPhone 16 Pro Max - 2026-XX-XX at HH.MM.SS.png
```

### 4-2. 解像度確認

ターミナルで：
```bash
sips -g pixelWidth -g pixelHeight ~/Desktop/Simulator\ Screenshot*.png
```

すべて **1290 × 2796** になっているか確認。違う数字なら違うサイズのシミュレータで撮ってしまっています（やり直し）。

### 4-3. ファイル名を整理

提出時に管理しやすいようリネーム：
```bash
cd ~/Desktop
mv "Simulator Screenshot - iPhone 16 Pro Max - "*"01"*.png ss-01-home.png
mv "Simulator Screenshot - iPhone 16 Pro Max - "*"02"*.png ss-02-plan.png
mv "Simulator Screenshot - iPhone 16 Pro Max - "*"03"*.png ss-03-session.png
mv "Simulator Screenshot - iPhone 16 Pro Max - "*"04"*.png ss-04-history.png
mv "Simulator Screenshot - iPhone 16 Pro Max - "*"05"*.png ss-05-share.png
```

## 5. 提出時の禁止事項チェックリスト

撮影後、以下を確認：

- [ ] **空画面のスクショがない**（必ず何かのデータが入っている）
- [ ] **実在の氏名・メールアドレス**が含まれていない
- [ ] **未実装機能の UI**（「Coming Soon」等）が映っていない
- [ ] **設定画面に外部 AI トグルが映っていない**（v1.0 では非表示）
- [ ] **「AI」「最強」「革命」「パーソナライズ」**などの禁止ワードが映っていない
- [ ] **5 枚の解像度が全て 1290 × 2796**（または許容サイズの 1260×2736 / 1320×2868）

## 6. App Store Connect へのアップロード

提出計画書 §5.4 の流れで、App Store Connect の以下に 5 枚をドラッグ：

```
My Apps → Muscle Mate → App Store タブ
  → iOS 1.0 Prepare for Submission
    → スクリーンショット セクション → iPhone 6.9" Display
```

5 枚アップロードすれば、6.9 inch の他サイズ枠（6.5 inch 等）にも自動転用されます。

## トラブルシューティング

### Q. Cmd + S が効かない
- シミュレータウィンドウがアクティブか確認
- メニューバー → File → New Screen Shot でも同じ機能

### Q. 6.9 inch シミュレータが見つからない
```bash
xcrun simctl list devices available | grep "Pro Max"
```
で利用可能なデバイスを確認。なければ Xcode → Settings → Platforms から iOS の最新 Simulator を追加。

### Q. アプリのデータをリセットしたい
シミュレータメニュー → Device → Erase All Content and Settings
